# DatePicker

Date pickers let users select a single date through either direct text input (masked to a configurable format) or a visual calendar popup. The `DatePickerBuilder` provides a reactive API for constructing date inputs with minimum/maximum date constraints, custom date formats, validation errors, and glass-effect styling — all driven by RxJS observables for live-updating bindings.

The date picker follows Material Design 3 conventions with a calendar grid popup, month navigation, and input masking that enforces the configured date format.

---

## When to use

Use a date picker when users need to select a specific calendar date:

- **Structured date entry** — the input mask guides the user to enter dates in the correct format (e.g. "DD/MM/YYYY")
- **Calendar visual selection** — the popup calendar helps users find dates relative to today or navigate between months
- **Date-range constraints** — with `withMinDate` and `withMaxDate` to restrict selectable dates

Use a plain text field for free-form date input when the format is not strict or when the date is entered programmatically (e.g. a system-generated timestamp).

---

## Usage

```typescript
import { DatePickerBuilder, DayOfWeek } from '@tdq/ora-components';
import { BehaviorSubject, of } from 'rxjs';

const value$ = new BehaviorSubject<Date | null>(null);

const picker = new DatePickerBuilder()
  .withCaption(of('Invoice Date'))
  .withValue(value$)
  .withFormat('DD/MM/YYYY')
  .withFirstDayOfTheWeek(DayOfWeek.MONDAY)
  .build();

document.body.appendChild(picker);

// Read the selected date
value$.subscribe(date => {
  console.log('Selected:', date?.toDateString());
});
```

## Builder API

All builder methods return `this` for chaining. Pass RxJS observables into the reactive setters — the component handles subscription lifecycle and automatically cleans up when detached from the DOM.

- **`withValue`** — `(value: Subject<Date \| null>): this`
  Binds a two-way `Subject` for the selected date. The component writes user input or calendar selections into the subject and reads incoming values to update the field.

- **`withCaption`** — `(caption: Observable<string>): this`
  Sets the label text displayed above the input. Updates reactively when the observable emits.

- **`withMinDate`** — `(min: Observable<Date>): this`
  Sets the minimum selectable date. Dates before this are disabled in the calendar.

- **`withMaxDate`** — `(max: Observable<Date>): this`
  Sets the maximum selectable date. Dates after this are disabled in the calendar.

- **`withFormat`** — `(format: string): this`
  Sets the date format string. Uses `D`, `M`, `Y` placeholders separated by delimiters. Default is `'DD-MM-YYYY'`.

- **`withEnabled`** — `(enabled: Observable<boolean>): this`
  Controls the interactive state. When `false` the input and calendar button are disabled with reduced opacity.

- **`withError`** — `(error: Observable<string>): this`
  Sets the validation error message shown below the input. An empty string clears the error state.

- **`withStyle`** — `(style: Observable<FieldStyle>): this`
  Sets the visual style variant: `FieldStyle.TONAL` (default) or `FieldStyle.OUTLINED`.

- **`withClass`** — `(className: Observable<string>): this`
  Appends additional CSS class names to the root container element.

- **`withFirstDayOfTheWeek`** — `(day: DayOfWeek): this`
  Sets which day appears first in the calendar grid. Defaults to `DayOfWeek.MONDAY`.

- **`asGlass`** — `(isGlass?: boolean): this`
  Enables the glass-effect variant with frosted background and blur. Defaults to `true`.

- **`build`** — `(): HTMLElement`
  Constructs and returns the root element. Subscribe to observables and wire up the DOM. Call once per builder. The returned element also exposes `showPopover()`, `hidePopover()`, and `toggle()` methods.

---

## Styling Variants

The date picker supports the same styling variants as `TextField`:

- **TONAL** (default) — Filled input with an active indicator at the bottom.
- **OUTLINED** — Bordered input with rounded corners.

## Keyboard

| Key | Action |
|---|---|
| `Tab` / `Shift+Tab` | Move focus into and out of the date picker |
| `Alt` + `Arrow Down` | Open the calendar popup |
| `Arrow keys` | Navigate between dates in the calendar grid |
| `Enter` | Select the focused date |
| `Escape` | Close the calendar popup |
| `Page Up` / `Page Down` | Navigate to previous / next month |

## Styling

All visual properties are driven by MD3 design tokens. Override them on `:root` or a scoped parent to restyle this component without touching source.

| CSS variable | Role |
|---|---|
| `--md-sys-color-surface` | Calendar popup background |
| `--md-sys-color-primary` | Selected date background |
| `--md-sys-color-on-primary` | Selected date text colour |
| `--md-sys-color-outline` | Input and popup border |
| `--md-sys-shape-medium` | Calendar popup border radius |

## Additional notes

- **Input height** is 48px (`h-[48px]`) for consistency with other form fields.
- **Calendar popup** is powered by `PopoverBuilder` and rendered at a fixed width of 320px.
- **Input masking** prevents typing characters that do not match the configured format. Separators (e.g. `/`, `-`) are auto-inserted when digits are typed at separator positions.
- **Value parsing** on manual input uses `parseDate` and only emits via `withValue` when the result is a valid date or empty.
- **Month navigation** uses `Icons.CHEVRON_LEFT` and `Icons.CHEVRON_RIGHT` in 20×20px wrappers.
- **Calendar icon** uses `Icons.CALENDAR` in a 24×24px wrapper and toggles the popup on click.
- **Disabled state** applies `opacity-38` and `pointer-events-none` to the container.
- **Error state** applies `border-error` and `focus-within:border-error` to the input wrapper.
- **Glass effect** overrides the input background with `glass-effect`, `rounded-small`, and translucent text colours.
- **Built element** exposes `showPopover()`, `hidePopover()`, and `toggle()` for external popup control.
